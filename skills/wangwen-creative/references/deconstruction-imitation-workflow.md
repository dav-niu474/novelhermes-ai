# 场景全流程拆文仿写创作指南

> 拆文是输入，仿写是输出。全流程拆文仿写是将"学"和"做"无缝衔接的完整工作流——从搜索下载一部优秀作品，到拆解它的创作原理，再到用这些原理创作出属于自己的新作品。

---

## 一、全流程概述

### 1.1 什么是"场景全流程拆文仿写"

场景全流程拆文仿写是一个**从读到写的一体化创作工作流**，它将以下能力串联成完整链路：

```
搜索下载 → 精读标注 → 六维度拆解 → 原理提取 → 仿写创作 → 素材入库
```

传统拆文和创作是割裂的——拆完就完了，到创作时已经忘了拆了什么。全流程拆文仿写解决了这个问题：拆文的每一步都指向创作，创作的每一步都引用拆文结果。

### 1.2 核心理念

> **拆文不是目的，仿写才是。拆文是为了理解原理，仿写是为了用原理创造出新东西。**

三个关键认知：

1. **拆文是逆向工程**：把别人的作品拆成零件，理解每个零件为什么这么设计
2. **原理是中间产物**：零件不能直接用（那是抄袭），要提炼出原理，原理才是可复用的
3. **仿写是正向工程**：用原理做蓝图，用自己的创意做材料，造出新的作品

### 1.3 与传统拆文的区别

| 维度 | 传统拆文 | 全流程拆文仿写 |
|---|---|---|
| **终点** | 输出拆文报告 | 输出新作品 |
| **分析粒度** | 全书宏观 | 场景级微观+全书宏观 |
| **输出物** | 拆文笔记 | 拆文笔记 + 创作原理 + 仿写作品 |
| **素材入库** | 可选 | 强制，每条原理必须入库 |
| **创作衔接** | 无 | 原理直接驱动创作 |
| **验证** | 无 | 仿写作品与原文的对比验证 |

### 1.4 触发条件

- 用户说"仿写XX"、"学着XX写一个"、"按XX的风格创作"→ 全流程拆文仿写
- 用户说"我想写出类似XX的感觉"→ 全流程拆文仿写
- 用户同时提到拆文和创作（"拆一下XX然后帮我写"）→ 全流程拆文仿写
- 用户说"拆文仿写"、"场景仿写"→ 全流程拆文仿写

### 1.5 什么时候不用全流程

- 用户只做拆文不做创作 → 仅走拆文流程（见 SKILL.md 拆文分析能力）
- 用户有明确设定直接要创作 → 走标准创作流程（见 SKILL.md 工作流程）
- 用户只是小修小补 → 不需要拆文仿写
- 用户要续写/改写自己已有的作品 → 不需要拆文仿写

## 二、六步全流程详解

### Step 1 · 搜索下载（获取原料）

> **目标**：获取原文，为后续分析提供精确素材。

**输入**：用户提供的书名/作者/URL/关键词
**输出**：本地存档的原文文件

**执行流程**：

1. 根据用户提供的信息，搜索并定位目标作品
2. 获取书籍元数据（书名、作者、字数、状态、标签、简介）
3. 根据仿写目的确定下载范围：
   - **学习开篇**：前3-5章
   - **学习节奏**：前10-30章
   - **学习特定场景**：包含目标场景的3-5章
   - **全面学习**：前50-100章（一般不需要全书）
4. 下载章节内容，本地存档

**关键决策点**：

| 仿写目标 | 下载范围 | 拆解重点维度 |
|---|---|---|
| 学开篇钩子 | 前3-5章 | 结构 + 文笔 |
| 学节奏控制 | 前10-30章 | 节奏 + 结构 |
| 学爽点设计 | 含大爽点的5-10章 | 节奏 + 角色 |
| 学人物塑造 | 含人物高光场景的5-10章 | 角色 + 文笔 |
| 学世界观搭建 | 前20-50章 | 设定 + 结构 |
| 学伏笔埋设 | 前50-100章 | 节奏 + 结构 |
| 全面仿写风格 | 前30-50章 | 全六维度 |

详细搜索下载操作 → `references/novel-search-download.md`

### Step 2 · 精读标注（建立直觉）

> **目标**：建立对原文的直觉感受，标记关键节点，为结构化拆解做准备。

**输入**：本地存档的原文
**输出**：带标注的原文副本 + 标注摘要

**标注符号系统**：

| 符号 | 含义 | 标注方式 |
|---|---|---|
| ★ | 爽点 | ★小爽/★中爽/★大爽 + 简述原因 |
| ◆ | 伏笔 | ◆埋设/◆回收 + 简述内容 |
| ▲ | 转折 | ▲ + 简述转折内容 |
| ● | 钩子 | ●悬念/●反转/●信息/●情感 + 简述 |
| ◎ | 名场面 | ◎ + 简述场景亮点 |
| ⚡ | 角色辨识度 | ⚡ + 简述什么让这个角色活了 |
| ? | 疑问 | ? + 不理解的地方 |
| ! | 精彩技法 | ! + 简述技法亮点 |

**标注原则**：
- **先标后析**：先快速标记，不在标注阶段做深度分析
- **简短感受**：每个标注10字以内，如"★中爽：打脸了，铺垫3章"
- **不跳过任何段**：逐段扫描，好作品处处有细节
- **标记直觉反应**：哪里让你心跳加速？哪里让你想跳过？哪里让你放不下？这些直觉比理性分析更有价值

**标注摘要输出**：

```markdown
## 精读标注摘要 · <书名>

> 标注日期：YYYY-MM-DD
> 标注范围：第X-Y章

### 标注统计
| 类型 | 数量 | 关键标记 |
|---|---|---|
| ★爽点 | N | 最强：第X章（★大爽） |
| ◆伏笔 | N | 最妙：第X章（◆埋设） |
| ▲转折 | N | 最意外：第X章 |
| ●钩子 | N | 最抓人：第X章末尾 |
| ◎名场面 | N | 最佳：第X章 |
| ⚡辨识度 | N | 最活：XXX角色 |
| ?疑问 | N | 最多疑问：XXX |
| !技法 | N | 最妙技法：XXX |

### 直觉记录
- 最抓我的段落：第X章，因为___
- 最想跳过的段落：第X章，因为___
- 最意外的转折：第X章，___
- 最想学的技法：___
```

### Step 3 · 六维度拆解（结构化分析）

> **目标**：将直觉感受转化为结构化的创作原理。

**输入**：带标注的原文 + 标注摘要
**输出**：六维度拆解报告

**拆解维度与仿写的对应关系**：

| 拆解维度 | 拆解内容 | 仿写产出 |
|---|---|---|
| **结构拆解** | 骨架如何搭建 | 我的作品用什么结构 |
| **角色拆解** | 角色为何有辨识度 | 我的角色怎么设计 |
| **设定拆解** | 世界观如何自洽 | 我的设定怎么构建 |
| **节奏拆解** | 爽点如何分布 | 我的节奏怎么安排 |
| **文笔拆解** | 技法为什么有效 | 我的文笔怎么修炼 |
| **商业拆解** | 读者为什么追读 | 我的市场怎么定位 |

每个维度的拆解必须遵循 **"原理三层次"** 提取法：

```markdown
### 原理提取模板

**原理名称**：<给原理取一个描述性的名字>

**表层（具体技巧）**：
- 在原文中是怎么做的：<描述>
- 可以直接模仿的手法：<描述>

**中层（技术原理）**：
- 为什么这么做有效：<解释底层机制>
- 这个原理的适用条件：<什么情况下有效>

**深层（创作哲学）**：
- 这个原理背后的创作哲学：<抽象到可跨类型复用>
- 如何在自己的创作中运用这个哲学：<描述>
```

六维度拆解详细方法 → `references/deconstruction-guide.md`

### Step 4 · 原理提取与入库（转化弹药）

> **目标**：将拆解结论转化为可直接驱动作创的"弹药"。

**输入**：六维度拆解报告
**输出**：结构化的创作原理卡片 + 素材入库

**原理卡片格式**：

```markdown
# 创作原理卡片

## 原理1：[名称]
- **来源**：<书名> · 第X章 · <维度>拆解
- **表层技巧**：<具体手法>
- **中层原理**：<为什么有效>
- **深层哲学**：<跨类型可复用的本质>
- **可复用方法**：<我怎么用>
- **适用场景**：<什么情况下用>
- **不适用场景**：<什么情况下不能用>
- **质量评分**：<独特性×30% + 可操作性×30% + 扩展性×20% + 类型契合度×20%> = 总分/10

## 原理2：[名称]
...
```

**入库质量门槛「5-10-2-8」**：

| 阶段 | 要求 | 理由 |
|---|---|---|
| **5轮精读** | 多角度交叉阅读 | 一遍扫过会遗漏细节 |
| **10条候选原理** | 从六维度凑10个备选 | 量不够选不出精品 |
| **精选2条入库** | 从10条里选最好的2条 | 全入库=品位稀释 |
| **每条8分以上** | 不够8分不入库 | 凑数7分入库降低整体质量 |

### Step 5 · 仿写创作（原理变现）

> **目标**：用提取的创作原理，创作出属于自己的新作品。

**输入**：创作原理卡片 + 用户的创作需求
**输出**：仿写作品（大纲 + 正文）

**仿写三步转化法**：

#### 第一步：模式识别

从原理卡片中筛选出与用户创作意图匹配的原理：

- **类型匹配**：升级流不适合用推理流的爽点配方
- **调性匹配**：轻松搞笑不适合用暗黑沉重的人设模板
- **能力匹配**：新手不适合用需要多线并行的结构模板

#### 第二步：参数调优

选定的原理不是直接套用的，需根据自己作品的特点调整参数：

- 赌注螺旋应该比模板快还是慢？（取决于目标读者群）
- 爽点密度应该比模板高还是低？（取决于更新频率和篇幅）
- 金手指限制应该比模板严还是松？（取决于故事风格）

#### 第三步：创意注入

原理提供骨架，创意才是血肉。在原理的框架内注入：

- **独特的世界观设定**：不是照搬原文的世界，而是用原理构建自己的世界
- **有辨识度的人物**：用原文的角色弧光原理，设计完全不同的角色
- **新鲜的爽点变体**：用原文的爽点配方原理，设计不同情境的爽点
- **个人化的语言风格**：不模仿原文的文笔，而是找到自己的声音

**仿写创作的执行纪律**：

1. **禁止照搬具体内容**：不能把原文的情节/人物/对话换个名字就用
2. **必须引用原理卡片**：仿写时注明用了哪条原理，原理是什么
3. **给变体不给唯一答案**：每个创作决策给出3+个变体选择
4. **Junior Writer模式**：先展示骨架和reasoning，等用户确认再填充

**仿写场景模板**：

仿写通常以**场景**为单位进行，而非整书。一个完整的场景仿写包含：

```markdown
# 场景仿写 · <场景类型>

## 参考场景
- 来源：<书名> · 第X章
- 场景概述：<50字概括参考场景>
- 使用的创作原理：<列出原理卡片ID>

## 仿写方案A：[方案名]
- 场景概述：<50字概括>
- 铺垫设计：<如何建立期待>
- 爽点设计：<如何释放满足感>
- 钩子设计：<如何留住读者>
- 开篇200字预览：

[仿写正文片段]

## 仿写方案B：[方案名]
...

## 仿写方案C：[方案名]
...
```

### Step 6 · 对比验证与素材入库（闭环）

> **目标**：验证仿写效果，确保"学到了"而非"抄了"，并将新素材入库。

**输入**：仿写作品 + 原文 + 原理卡片
**输出**：验证报告 + 新增素材入库

#### 对比验证

仿写完成后，必须进行对比验证——不是对比"像不像原文"，而是对比"原理有没有生效"：

| 验证维度 | 验证方法 | 通过标准 |
|---|---|---|
| **结构验证** | 仿写的结构是否遵循了提取的结构原理 | 结构清晰、节奏到位、钩子有效 |
| **节奏验证** | 仿写的爽点分布是否符合节奏原理 | 爽点密度合理、无疲劳段、钩子密集 |
| **角色验证** | 仿写的角色是否有原文级别的辨识度 | 去掉名字能认出是谁 |
| **设定验证** | 仿写的设定是否自洽 | 核心规则可推导、无逻辑漏洞 |
| **原创性验证** | 仿写与原文的差异度 | 换个世界观故事仍然成立 |

**原创性验证是最关键的**。判断标准：

- ✅ **学到了原理**：用原文的节奏原理，写出完全不同的故事
- ✅ **跨类型复用**：用修仙文的爽点原理，写出都市文的爽点
- ❌ **只是换皮**：把萧炎换成李炎，把斗气换成灵气
- ❌ **照搬情节**：把退婚改成退约，但情节走向完全一样

#### 素材入库

仿写过程中产生的优质素材必须入库：

1. **仿写中产生的新原理**：创作时对原原理的新理解/改进
2. **仿写中产生的废案**：废弃方案中好的创意碎片
3. **仿写中发现的问题**：转化为"避坑素材"
4. **仿写中的原创片段**：好的原创文字可作为"文笔参考"

#### 原文处理

仿写验证通过后：
- 保留拆文笔记和原理卡片
- 保留仿写作品
- **删除原文**（仅用于学习，不再需要）

## 三、场景级仿写实战模板

### 3.1 开篇场景仿写

**参考目标**：学习优秀开篇的钩子设计

**拆解重点**：
- 开篇钩子类型（危机钩/悬念钩/反差钩/金手指钩）
- 3000字内的信息密度
- 第一章结尾的钩子设计

**仿写模板**：

```markdown
# 开篇仿写

## 参考分析
- 原文开篇类型：[钩子类型]
- 原文3000字内交付了：[列出信息点]
- 原文第一章结尾钩子：[描述]

## 创作原理
- 原理ID：XXX
- 核心原理：[描述]
- 参数调优：[与原文的差异化]

## 仿写输出
### 方案A：[名称] - [钩子类型]
[2000-3000字开篇]

### 方案B：[名称] - [钩子类型]
[2000-3000字开篇]

### 方案C：[名称] - [钩子类型]
[2000-3000字开篇]
```

### 3.2 爽点场景仿写

**参考目标**：学习爽点设计的四段式结构

**拆解重点**：
- 铺垫方式（建立期待的手法）
- 触发条件（什么引爆了爽感）
- 爆发形式（爽感如何释放）
- 余韵处理（爽感如何延长）

**仿写模板**：

```markdown
# 爽点场景仿写

## 参考分析
- 原文爽点类型：[打脸/升级/获宝/翻盘/碾压/解谜/反转/情绪爆发]
- 四段式比例：铺垫___% → 触发___% → 爆发___% → 余韵___%
- 铺垫手法：[描述]
- 触发方式：[描述]
- 爆发手法：[描述]
- 余韵方式：[描述]

## 创作原理
- 原理ID：XXX
- 参数调优：[调整哪些参数]

## 仿写输出
### 方案A：[爽点变体描述]
- 铺垫：[3-5句描述]
- 触发：[1-2句描述]
- 爆发：[3-5句描述]
- 余韵：[2-3句描述]
[完整场景]

### 方案B...
```

### 3.3 伏笔场景仿写

**参考目标**：学习伏笔的埋设和回收技法

**拆解重点**：
- 埋设方式（显性/隐性/反向/嵌套）
- 埋设时的伪装度
- 回收时的效果类型
- 埋收间隔和节奏

**仿写模板**：

```markdown
# 伏笔场景仿写

## 参考分析
- 原文伏笔层级：[主线/支线/细节]
- 伏笔技术：[显性/隐性/反向/嵌套]
- 埋设方式：[描述]
- 回收效果：[恍然大悟/震惊/感动/恐惧]
- 读者可察觉度：[高/中/低]

## 创作原理
- 原理ID：XXX
- 参数调优：[调整哪些参数]

## 仿写输出
[包含埋设和回收的完整场景设计]
```

### 3.4 对话场景仿写

**参考目标**：学习对话的辨识度和潜台词设计

**拆解重点**：
- 对话四层次：信息层/情绪层/性格层/潜台词层
- 去名识别度（不看名字能否认出是谁在说话）
- 对话节奏（长短交替/打断/沉默）

**仿写模板**：

```markdown
# 对话场景仿写

## 参考分析
- 原文对话风格：[描述]
- 辨识度来源：[语气词/句式/口头禅/信息偏好]
- 潜台词案例：[对话表面含义 vs 实际含义]

## 创作原理
- 原理ID：XXX

## 仿写输出
[3个角色的对话场景，每个角色有独特的对话指纹]
```

### 3.5 反转场景仿写

**参考目标**：学习反转设计的信息控制和预期管理

**拆解重点**：
- 铺垫阶段给读者的"错误预期"
- 信息控制（读者知道什么、不知道什么、以为知道但其实是错的）
- 反转触发点
- 反转后的余波处理

**仿写模板**：


## 四、仿写与抄袭的红线

### 4.1 明确的判断标准

| 行为 | 仿写 ✅ | 抄袭 ❌ |
|---|---|---|
| 提取节奏原理，用自己的情节实现 | ✅ | |
| 提取爽点配方，在不同类型中实现 | ✅ | |
| 提取角色弧光原理，设计不同背景的角色 | ✅ | |
| 把原文的情节换个名字重新讲一遍 | | ❌ |
| 把原文的对话稍微改几个字就复制 | | ❌ |
| 把原文的世界观换个皮照搬 | | ❌ |
| 把原文的人物关系整体移植 | | ❌ |

### 4.2 "三换原则"验证法

仿写完成后，用"三换原则"验证原创性：

1. **换世界观**：把你的故事放到完全不同的世界里，情节是否仍然成立？
   - 成立 → 你学到的是原理 ✅
   - 不成立 → 你照搬的是情节 ❌

2. **换类型**：把你的爽点放到不同类型的作品里，是否仍然有效？
   - 有效 → 你学到的是原理 ✅
   - 无效 → 你依赖的是特定类型套路 ❌

3. **换角色**：把你的角色换成完全不同的性格，场景是否仍然精彩？
   - 精彩 → 你学到的是原理 ✅
   - 不精彩 → 你依赖的是特定角色特质 ❌

### 4.3 仿写声明

每次仿写完成，应在作品头部添加仿写声明：

```markdown
> 本作品为拆文仿写练习，学习参考了《XXX》的[具体维度]创作原理。
> 使用的核心原理：[原理名称列表]
> 本作品为原创内容，仅学习方法论，未使用原文的具体情节/人物/对话。
```

## 五、全流程时间线参考

| 阶段 | 30章小说 | 10章短篇 | 单场景仿写 |
|---|---|---|---|
| 搜索下载 | 5-10分钟 | 2-3分钟 | 1-2分钟 |
| 精读标注 | 60-90分钟 | 20-30分钟 | 10-15分钟 |
| 六维度拆解 | 90-120分钟 | 40-60分钟 | 20-30分钟 |
| 原理提取入库 | 30-45分钟 | 15-20分钟 | 10-15分钟 |
| 仿写创作 | 60-120分钟 | 30-60分钟 | 20-40分钟 |
| 对比验证 | 20-30分钟 | 10-15分钟 | 5-10分钟 |
| **总计** | **4.5-6.5小时** | **2-3小时** | **1-1.5小时** |

**建议策略**：
- 新手从**单场景仿写**开始，熟练后再做10章短篇仿写
- 每次仿写重点学1-2个维度，不要贪多
- 同一作品可以多次仿写，每次聚焦不同维度

## 六、常见问题

### Q1：仿写出来的东西总是很像原文怎么办？

这说明你停留在"表层技巧"层面，没有提取到"深层原理"。解决方案：
- 回到原理提取步骤，确保每条原理都回答了"为什么有效"
- 用"三换原则"验证你的原理是否跨类型有效
- 尝试用完全不同的世界观/类型去实现同一个原理

### Q2：拆文很详细但仿写时用不上怎么办？

这说明拆文和仿写脱节了。解决方案：
- 拆解时每一条结论都必须回答"这条结论能帮我做什么"
- 仿写时强制引用原理卡片，注明用了哪条原理
- 从单场景仿写开始，不要一上来就仿写整书

### Q3：仿写时创意不足怎么办？

仿写的创意不足通常是因为"参数调优"和"创意注入"步骤做得不够。解决方案：
- 在"创意注入"步骤刻意做"类型交叉"——用修仙文的结构原理写都市文
- 给每个创作决策至少3个变体，不要第一个方案就定死
- 参考素材库中的其他素材做"组合创新"——A的结构 + B的爽点 + C的文笔

### Q4：全流程太长了，有没有快速路径？

有。根据目的选择：

| 目的 | 快速路径 | 省略的步骤 |
|---|---|---|
| 快速学一个技法 | 精读标注 → 单维度拆解 → 原理提取 → 仿写片段 | 跳过五维度、跳过入库 |
| 快速学开篇 | 下载前3章 → 结构+文笔拆解 → 开篇仿写 | 跳过其他维度 |
| 快速学爽点 | 下载含爽点的3章 → 节奏拆解 → 爽点仿写 | 跳过其他维度 |

快速路径的代价是**原理不够深**，适合"先用起来"，后续再补全。
