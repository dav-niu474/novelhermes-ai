# 自检清单 · 交付前的质量守门

> 每次交付前用这份清单过一遍。不是走流程，是真的要检查。
> 读者不会因为一个bug停止阅读，但会因为十个bug弃书。

## 一、内容自检

### 1.1 设定一致性

- [ ] novel-spec.md 中的金手指在正文中使用方式一致
- [ ] 升级体系的每次突破都在 spec 的条件下
- [ ] 世界观规则没有自相矛盾
- [ ] 人物性格没有突变（除非有剧情原因）
- [ ] 时间线没有错乱

### 1.2 伏笔管理

- [ ] 伏笔追踪表已更新
- [ ] 没有超过预警期的活跃伏笔
- [ ] 本卷新埋的伏笔有明确的回收计划
- [ ] 已回收的伏笔在追踪表中标注了实际回收章节

### 1.3 剧情逻辑

- [ ] 每个重要事件都有合理的因果链
- [ ] 角色的重大决定有充分动机
- [ ] 反派的行为有逻辑（不是纯为了搞主角）
- [ ] 巧合只在推动剧情时使用，不用于解决危机

## 二、节奏自检

### 2.1 三定律检查

- [ ] 每3000字至少一个小钩子（悬念/反转/信息揭露）
- [ ] 每3章至少一个大爽点
- [ ] 每10章完成一个小剧情弧

### 2.2 章节钩子

- [ ] 每章结尾都有钩子（让读者想看下一章）
- [ ] 钩子类型有变化（不是每章都是"悬念"）
- [ ] 钩子强度有起伏（不是每章都是"大悬念"）

### 2.3 信息密度

- [ ] 没有超过2章的纯铺垫无回报
- [ ] 没有超过5章的纯日常无推进
- [ ] 对话场景有信息增量（不是纯闲聊）
- [ ] 描写场景有叙事功能（不是纯装饰）

## 三、可读性自检

### 3.1 文字质量

- [ ] 没有大段无断落的文字墙
- [ ] 对话有角色辨识度（蒙住名字能分辨是谁在说话）
- [ ] 没有过于复杂的从句（一句话说清一个意思）
- [ ] 关键场景有画面感

### 3.2 读者体验

- [ ] 开篇3章能抓住目标读者
- [ ] 没有需要读者往前翻才能理解的段落
- [ ] 关键信息的交代自然（不是NPC解说员式）
- [ ] 情感高潮有铺垫（不是突然煽情）

## 四、金手指自检

- [ ] 金手指的使用在 spec 的框架内
- [ ] 金手指没有出现未解释的新能力
- [ ] 金手指的限制机制在正文中体现
- [ ] 金手指的成长有节奏（不是突然变强）
- [ ] 金手指的展示频率合理（不嫌多不嫌少）

## 五、连载可持续性自检（长篇专属）

- [ ] 当前进度的主线矛盾有升级空间
- [ ] 备用剧情线至少有2条可启用
- [ ] 配角有独立故事线可展开
- [ ] 世界观有未探索的区域/层级
- [ ] 升级体系还有上升空间
- [ ] 伏笔有跨卷的长线布局

## 自检评分

| 维度 | 权重 | 得分(0-10) | 加权 |
|---|---|---|---|
| 设定一致性 | 25% | | |
| 节奏把控 | 25% | | |
| 可读性 | 20% | | |
| 金手指规范 | 15% | | |
| 连载可持续 | 15% | | |
| **综合分** | 100% | | |

**交付门槛**：综合分 ≥ 7/10。低于7分的部分需要修改后再交付。
