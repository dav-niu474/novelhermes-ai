# 小说搜索下载指南

> 拆文仿写的第一步是拿到原文。本指南描述如何搜索和下载番茄小说平台上的作品，为后续拆文分析提供原始素材。

---

## 一、能力概述

### 1.1 为什么需要搜索下载

拆文仿写创作流程的起点是**获取原文**。仅有书名和模糊印象是不够的——精确的拆文分析需要逐章精读、逐段标注，这要求我们能：

- **搜索定位**：按书名/作者/类型在番茄小说平台上找到目标作品
- **获取信息**：提取作品的元数据（书名、作者、字数、状态、标签、简介）
- **下载章节**：批量获取作品章节目录和正文内容
- **本地存档**：将原文保存为结构化的本地文件，方便后续标注和拆解

### 1.2 触发条件

- 用户说"帮我下载XX"、"搜一下XX这本小说"、"找到XX的原文"→ 进入搜索下载模式
- 用户要求对某本具体作品做拆文分析 → 先搜索下载原文
- 用户说"想看XX前几章是怎么写的"→ 搜索下载指定章节
- 创作仿写需要参考原文时 → 自动触发搜索下载

### 1.3 版权与使用边界

| 行为 | 允许 | 禁止 |
|---|---|---|
| 搜索公开可见的小说基本信息 | ✅ | |
| 获取章节目录列表 | ✅ | |
| 下载免费章节用于个人学习拆文 | ✅ | |
| 下载VIP/付费章节 | | ❌ |
| 将下载内容用于商业用途 | | ❌ |
| 将下载内容分发/传播 | | ❌ |
| 下载内容仅用于拆文学习，拆完后删除原文 | ✅ | |

**核心原则：下载的原文仅供拆文学习使用，学习完毕后应删除原文，仅保留拆文笔记和提取的创作原理。**

## 二、番茄小说平台 API

### 2.1 平台概述

番茄小说（fanqienovel.com）是字节跳动旗下的免费网文平台，拥有海量作品资源。其网页版提供了可访问的 API 接口，可以获取作品信息和章节内容。

### 2.2 核心 API 端点

#### 获取书籍信息

```
GET https://fanqienovel.com/page/{book_id}
```

- **book_id**：书籍唯一标识，纯数字字符串（如 `7423591956359416856`）
- **返回**：HTML 页面，包含书籍元数据（嵌入在 `__NEXT_DATA__` 或 `__INITIAL_STATE__` 的 JSON 中）
- **可提取字段**：
  - `bookName` / `book_name` — 书名
  - `author` / `authorName` — 作者
  - `abstract` / `description` — 简介
  - `thumb_url` / `cover_url` — 封面图 URL
  - `chapterCount` / `chapter_count` — 章节总数
  - `tags` / `tagNames` — 标签
  - `status` / `serial_status` — 完结状态（0=连载中, 1=已完结）

#### 获取章节目录

```
GET https://fanqienovel.com/api/reader/directory/detail?bookId={book_id}
```

- **返回**：JSON 格式的章节列表
- **响应结构**：

```json
{
  "code": 0,
  "data": {
    "chapterListWithVolume": [
      [
        {"itemId": "xxx", "title": "第1章 xxx"},
        {"itemId": "yyy", "title": "第2章 xxx"}
      ]
    ]
  }
}
```

- **注意**：此接口有频率限制，建议请求间隔 ≥ 0.8秒
- **首次请求可能返回 403**：需要先访问书籍页面获取 Cookie，再重试

#### 获取章节内容

章节内容的获取需要通过第三方 API 服务（因为番茄官方 API 需要加密签名）。第三方 API 的通用端点格式为：

```
GET {third_party_base_url}/reading/reader/batch_full/v?item_ids={ids}&aid=1967&device_platform=android
```

- **item_ids**：章节 ID 列表，逗号分隔（如 `id1,id2,id3`）
- **aid**：固定值 `1967`
- **支持批量**：可一次请求 15-25 个章节
- **epub 模式**：添加 `&epub=1` 参数可获取更适合阅读的格式

### 2.3 Book ID 解析

用户可能以多种格式提供书籍标识，需要统一解析为 book_id：

| 输入格式 | 示例 | 解析方式 |
|---|---|---|
| 纯数字 ID | `7423591956359416856` | 直接使用 |
| 书籍页面 URL | `https://fanqienovel.com/page/7423591956359416856` | 正则提取 `/page/(\d+)` |
| 分享链接（含 query） | `...?book_id=7423591956359416856` | 正则提取 `book_id=(\d+)` |
| 短链接 | `https://changdunovel.com/t/E_HDbOHpMJA/` | 跟随重定向获取最终 URL，再解析 |

### 2.4 请求规范

为了保护平台服务和避免被封禁，搜索下载时必须遵守以下规范：

| 规范项 | 要求 |
|---|---|
| **User-Agent** | 使用标准浏览器 UA |
| **请求间隔** | 目录 API ≥ 0.8秒/次，内容 API ≥ 1秒/次 |
| **重试策略** | 最多 3 次，指数退避（0.6s → 1.2s → 2.4s） |
| **403 处理** | 先访问书籍页面预热 Cookie，再重试 |
| **批量大小** | 每次请求不超过 25 个章节 ID |
| **缓存** | 成功获取的目录数据本地缓存，避免重复请求 |

## 三、搜索下载工作流

### 3.1 标准流程

```
1. 识别输入 → 2. 搜索/解析 → 3. 获取书籍信息 → 4. 获取章节目录 → 5. 下载章节内容 → 6. 本地存档
```

### Step 1 · 识别输入

判断用户提供的是：
- **具体书名**（如"斗破苍穹"）→ 需要搜索找到 book_id
- **URL/链接**（如 fanqienovel.com/page/xxx）→ 直接解析 book_id
- **book_id**（纯数字）→ 直接使用
- **作者名**（如"天蚕土豆"）→ 搜索作者作品列表

### Step 2 · 搜索/解析

**情况A：有 book_id 或 URL** → 直接解析为 book_id

**情况B：只有书名/作者名** → 需要通过搜索功能找到 book_id
- 在番茄小说网站搜索书名
- 从搜索结果页面提取匹配作品的 book_id 和基本信息
- 如果有多条匹配结果，列出供用户选择

### Step 3 · 获取书籍信息

使用 book_id 请求书籍页面，提取元数据：

```markdown
## 书籍信息

| 字段 | 内容 |
|---|---|
| 书名 | |
| 作者 | |
| 类型 | |
| 字数 | |
| 章节数 | |
| 状态 | 连载中/已完结 |
| 标签 | |
| 简介 | |
```

向用户确认：是否下载此书？下载哪些章节？

### Step 4 · 获取章节目录

请求章节目录 API，返回完整的章节列表。展示给用户：

```markdown
## 章节目录

| 序号 | 章节标题 | 章节ID |
|---|---|---|
| 1 | 第1章 xxx | xxx |
| 2 | 第2章 xxx | xxx |
| ... | ... | ... |
```

让用户选择下载范围：
- **全部章节**
- **前N章**（拆文通常只需前10-30章）
- **指定章节范围**（如第5-15章）
- **指定章节列表**（如前3章 + 某个高潮段落）

### Step 5 · 下载章节内容

批量下载用户选定的章节内容：
- 每批 15-25 章
- 批次间间隔 1 秒
- 支持断点续传（已下载的章节跳过）
- 下载失败自动重试 3 次

### Step 6 · 本地存档

将下载的内容保存为结构化文件：

```
作品名/
├── book-info.md          # 书籍元数据
├── chapters/
│   ├── 001-第1章xxx.md   # 单章内容
│   ├── 002-第2章xxx.md
│   └── ...
└── full-text.txt         # 完整纯文本（可选）
```

**单章内容格式**：

```markdown
# 第X章 章节标题

> 章节ID: xxx
> 字数: xxx

[章节正文内容]
```

## 四、搜索下载与拆文仿写的整合

### 4.1 自动衔接流程

搜索下载不是独立功能，它是拆文仿写的前置步骤：

```
搜索下载 → 精读标注 → 六维度拆解 → 原理提取 → 仿写创作 → 素材入库
           ↑                                            │
           └──── 仿写完成，原文可删除 ←───────────────────┘
```

### 4.2 下载策略与拆文目的匹配

| 拆文目的 | 下载范围 | 理由 |
|---|---|---|
| 学习开篇钩子 | 前3-5章 | 钩子效果在前3章内完成 |
| 学习节奏控制 | 前10-30章 | 足够观察一个完整的节奏周期 |
| 学习伏笔体系 | 前50-100章 | 伏笔埋收需要足够的跨度 |
| 全面拆文 | 全书 | 完整拆解需要全局视角 |
| 单场景仿写 | 包含目标场景的3-5章 | 上下文足够理解场景设计 |

### 4.3 下载与精读标注的衔接

下载完成后，立即进入精读标注阶段：

1. 将原文加载到工作区
2. 使用标注符号逐段标注（★◆▲●◎⚡?!）
3. 标注完成后进入六维度拆解
4. 拆解完成后原文可归档或删除

## 五、常见问题

### Q1：搜不到想找的书怎么办？

可能原因：
- 书名输入有误 → 尝试模糊搜索或输入作者名
- 该书不在番茄小说平台 → 告知用户，建议换平台或手动提供原文
- 书名有特殊字符 → 尝试去掉特殊字符搜索

### Q2：下载内容有缺失章节怎么办？

- 先重试缺失章节（最多3次）
- 如果重试仍失败，标记为 `[本章下载失败]`
- 告知用户哪些章节下载失败，不影响已下载章节的拆文分析

### Q3：搜索下载的速度限制？

- 目录 API：≥ 0.8秒/请求
- 内容 API：≥ 1秒/请求
- 一本30章的小说，完整下载约需 2-3 分钟
- 一本300章的小说，完整下载约需 15-20 分钟

### Q4：下载的原文可以保留多久？

- 拆文学习期间可保留
- 拆文完成、原理提取入库后，建议删除原文
- 仅保留拆文笔记和提取的创作原理
- 这既是版权尊重，也避免素材库被原文堆满

## 六、技术参考

本指南的搜索下载能力参考了 [Tomato-Novel-Downloader](https://github.com/zhongbai2333/Tomato-Novel-Downloader) 项目的实现。该项目的核心架构包括：

| 模块 | 功能 | 对应文件 |
|---|---|---|
| **网络层** | API 请求、Cookie 管理、重试退避 | `src/network_parser/network.rs` |
| **内容解析** | HTML/JSON 解析、书籍信息提取 | `src/book_parser/parser.rs` |
| **章节下载** | 批量下载、断点续传、多线程 | `src/download/downloader.rs` |
| **ID 解析** | URL/短链/纯数字 → book_id | `src/base_system/book_id.rs` |
| **第三方 API** | 第三方内容服务对接 | `src/third_party/content_client.rs` |
| **输出格式** | TXT/EPUB/PDF 生成 | `src/book_parser/finalize_*.rs` |

关键 API 调用参数：
- `aid=1967`（应用 ID，固定值）
- `device_platform=android`
- `item_ids` 为逗号分隔的章节 ID
- 批量下载每组 15-25 章
- 支持 `epub=1` 参数获取阅读优化格式
