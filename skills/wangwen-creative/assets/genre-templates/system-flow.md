# 系统升级流 · 类型模板

> "叮！恭喜宿主获得XX能力。"

## 类型定位

- **核心卖点**：系统辅助+任务驱动+即时反馈+数据化成长
- **读者画像**：16-30岁，喜欢游戏化体验和即时满足感
- **创作难度**：⭐⭐⭐（系统设计是核心，设计不好全书崩）

## 推荐金手指

| 类型 | 系统设计 | 限制机制 | 适用变体 |
|---|---|---|---|
| 签到系统 | 每日签到获得奖励 | 高级奖励需要连续签到 | 修炼签到/生活签到/战斗签到 |
| 任务系统 | 完成系统任务获得奖励 | 任务失败有惩罚 | 主线任务/日常任务/隐藏任务 |
| 商城系统 | 用积分兑换物品/能力 | 积分获取有上限，高级物品天价 | 技能商城/装备商城/知识商城 |
| 模拟系统 | 可进入模拟空间练习 | 模拟有时间限制和次数限制 | 战斗模拟/人生模拟/修炼模拟 |

## 默认升级体系

系统流的升级是数据化的，每个等级有明确的数值区间：

```
F级（1-20章）· 普通人
  → E级（20-50章）· 超越常人
    → D级（50-100章）· 区域强者
      → C级（100-200章）· 城市级
        → B级（200-350章）· 省级
          → A级（350-500章）· 国家级
            → S级（500-700章）· 世界级
              → SS级（700-900章）· 超越人类
                → SSS级（900+章）· 神话级
```

**升级设计要点**：
- 每级有明确的数值范围和代表战力
- 跨级难度指数级增长
- 系统任务难度匹配当前等级
- 突破大等级时系统解锁新功能

## 典型节奏模板

| 阶段 | 章节范围 | 核心事件 | 节奏 |
|---|---|---|---|
| 绑定系统 | 1-3章 | 获得系统，了解规则 | 快速建立世界观 |
| 新手任务 | 3-15章 | 完成系统新手任务，F→E | 爽点密集，展示系统玩法 |
| 系统升级 | 15-30章 | 系统解锁新功能 | 新鲜感+爽点 |
| 对抗 | 30-50章 | 遭遇其他能力者/系统持有者 | 冲突升级 |
| 隐藏任务 | 50-80章 | 触发系统隐藏任务线 | 剧情深化 |
| 系统真相 | 80+章 | 逐步揭示系统来源 | 悬念+伏笔 |

## 开篇钩子推荐

### 钩子1：意外绑定（经典）
> 主角意外获得系统 → 探索系统功能 → 第一个任务完成获得爽感

### 钩子2：绝境触发（紧张）
> 主角陷入绝境 → 系统激活 → 绝地翻盘

### 钩子3：选择型系统（新颖）
> 系统给出选择，不同选择导向不同路线 → 读者的选择参与感

## 常见坑点与避坑

| 坑点 | 表现 | 避坑方法 |
|---|---|---|
| 系统太强 | 系统解决一切，主角没有主观能动性 | 系统提供工具，主角负责策略 |
| 刷怪无聊 | 重复做任务升级 | 任务要有故事性，不是纯刷 |
| 系统通知刷屏 | 每章大量系统提示 | 系统提示精简化，有叙事功能 |
| 系统来源不明 | 写了几百章还没解释系统是什么 | 至少在50章内给暗示，200章内给初步答案 |
