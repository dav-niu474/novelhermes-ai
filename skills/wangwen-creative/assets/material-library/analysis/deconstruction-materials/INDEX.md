# 拆文来源素材库

> **拆文来源素材单独存放。** 从具体作品拆文中提取的素材，与原创素材分开管理，便于溯源、版权管理和素材验证。

## 为什么拆文素材要独立存放

| 原因 | 说明 |
|------|------|
| **版权溯源** | 拆文素材来源于具体作品，需标注来源，便于版权管理 |
| **质量验证** | 拆文素材需经过"跨作品验证"才能升级为"验证原理"，独立存放便于标记验证状态 |
| **素材分层** | 原创素材（通用性高）vs 拆文素材（来源特定），分开管理避免混淆 |
| **按需清除** | 拆文完成学习后，原文可删除，拆文素材独立管理不受影响 |
| **反向查找** | 从拆文素材可追溯到来源作品和拆文报告 |

## 拆文素材格式

每条拆文素材必须包含以下字段（比原创素材多"来源作品"和"验证状态"）：

```markdown
### [ID]: [素材名称]
- **来源作品**：<书名> · <作者> · 第X-Y章（必填，拆文素材的溯源依据）
- **拆文日期**：YYYY-MM-DD
- **拆文维度**：<结构/角色/设定/节奏/文笔/商业>
- **原理三层次**：
  - 表层技巧：<具体手法>
  - 中层原理：<为什么有效>
  - 深层哲学：<跨类型可复用的本质>
- **具体描述**：<详细内容>
- **适用类型**：<类型列表>
- **验证状态**：<待验证 / 单作品验证 / 跨作品验证 / 通用原理>
- **质量评分**：<独特性×30% + 可操作性×30% + 扩展性×20% + 类型契合度×20%> = 总分/10
- **扩展方向**：<可能的变体>
```

## 拆文素材验证升级机制

拆文素材不是一成不变的，随着拆文作品增多，素材的"验证等级"会提升：

| 验证状态 | 条件 | 可信度 | 是否可合并到原创素材库 |
|---------|------|--------|---------------------|
| **待验证** | 仅从1部作品提取 | 低 — 可能是作者个人风格 | ❌ |
| **单作品验证** | 从1部作品提取 + 回写测试通过 | 中 — 该作品中有效 | ❌ |
| **跨作品验证** | 从3+部作品中观察到相同模式 | 高 — 类型级有效 | ✅ 可升级 |
| **通用原理** | 从5+部不同类型作品中验证 | 极高 — 跨类型有效 | ✅ 强烈建议升级 |

**升级流程**：
1. 拆文素材首次入库 → 标记"待验证"
2. 用该素材进行仿写创作，效果好 → 升级为"单作品验证"
3. 拆文3+部同类型作品，同一模式反复出现 → 升级为"跨作品验证"
4. 跨作品验证的素材 → 复制到对应原创素材目录，分配原创ID，标注"由XX-DNNN升级"
5. 原拆文素材保留，但标记为"已升级→XX-NNN"

## 拆文素材与原创素材的流转

```
拆文分析 → 提取素材 → 入库(deconstruction-materials/)
                        ↓
              验证(回写测试/跨作品验证)
                        ↓
              验证通过 → 复制到原创素材库(对应catalog.md)
                        ↓
              原拆文素材标记"已升级→新ID"
```

**关键规则**：
- 拆文素材不会自动进入原创素材库，必须经过验证
- 验证升级是单向的：拆文素材 → 原创素材，不会反向
- 已升级的拆文素材保留在拆文库中，但标记升级状态，保持溯源能力
- 创作时优先使用原创素材库，拆文素材库作为"实验性素材"参考使用
